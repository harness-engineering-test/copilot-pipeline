## 改善提案（優先度順）

### 優先度 1: backend/ の骨格実装（アーキテクチャ・コード品質・テスト）

`copilot-instructions.md` に定義された規約を実装に反映した骨格プロジェクトを作成する。

- `backend/` ディレクトリを作成し `go.mod` を初期化する
- resolver / service / repository の 3 層構成でパッケージを切る
- `backend/utils/ptr.go` をはじめとする共有ユーティリティを実装する
- 各レイヤーのサンプル実装と対応する `_test.go` を同時に追加する

### 優先度 2: セキュリティ基盤の整備

- JWT / セッション認証ミドルウェアを実装する
- 入力バリデーションライブラリ（`go-playground/validator` 等）を導入する
- シークレットは環境変数または Secret Manager 経由で注入し、コードにハードコードしない方針を CI で強制する

### 優先度 3: 運用性・可観測性の導入

- 構造化ログ（`zerolog` / `zap`）を導入し、リクエスト ID をすべてのログに付与する
- `/healthz` エンドポイントを実装し、Kubernetes Liveness/Readiness Probe に対応させる
- `golangci-lint` を CI ステップに組み込み、全 PR でリントが通ることを必須化する
